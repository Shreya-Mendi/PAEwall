System

How It Works

Four-Module Pipeline

A
Patent Intake Parse claims, extract independent limitations
→
B
Retrieval Engine BM25 / TF-IDF+LR / Dual-Encoder + FAISS · ranks 522 companies
→
C
Claim Chart LLM maps each limitation to product evidence, scored for faithfulness
→
D
Red Team Non-infringement arguments, invalidity (§101/102/103/112), enforcement probability

PAE-Bench Dataset

522patent–defendant pairs
5technology verticals
483unique patents
3public data sources
  • CourtListener RECAP Federal litigation dockets (nature of suit 830 = patent). Ground-truth labels: actual defendants named in court.
  • Google Patents BigQuery Full patent text, independent claims, CPC classification codes — matched by patent number.
  • SEC EDGAR 10-K Item 1 product descriptions for defendant companies, retrieved via company_tickers.json CIK lookup.

Train/test split by patent_id to prevent leakage. Positives = named defendants; negatives = other companies in the corpus.
